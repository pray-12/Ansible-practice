**Jinja2** is used to insert variables into files dynamically.

Instead of hardcoding configs → you generate them.

## Why use Jinja2?
- ❌ Static files
- ❌ Manual updates
- ❌ Not reusable
👉 With Jinja2
- ✅ Dynamic templates
- ✅ Reusable configs
- ✅ Easy maintenance

Example:

This is a Jinja2 template for an nginx configuration file (nginx.conf.j2):

```bash
server {
    listen 80;
    server_name {{ domain_name }};

    location / {
        proxy_pass http://{{ backend_ip }}:{{ backend_port }};
    }
}
```

Playbook to use this template:

```yaml
- hosts: web
  vars:
    domain_name: example.com
    backend_ip: 10.0.0.5
    backend_port: 8080

  tasks:
    - name: Deploy nginx config
      template:
        src: nginx.conf.j2
        dest: /etc/nginx/nginx.conf
```

Output:

```bash
server {
    listen 80;
    server_name example.com;

    location / {
        proxy_pass http://10.0.0.5:8080;
    }
}
```

## Real Use:
