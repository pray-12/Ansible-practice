In Ansible, cloud integration means using Ansible to create, manage, and automate cloud infrastructure.

## What It Means (In Simple Words)

Instead of manually creating resources in cloud consoles, you can use Ansible to automate:

- Create servers (VMs)
- Configure networking
- Deploy applications
- Manage scaling

## Example Use Case: AWS EC2

```yaml
- name: Create EC2 instance
  hosts: localhost
  connection: local
  tasks:
    - name: Launch instance
      ec2:
        key_name: mykey # SSH key for access
        instance_type: t2.micro # Type of VM
        image: ami-123456 # AMI ID for OS
        region: ap-south-1 # AWS region
        wait: yes # Wait until instance is running
        count: 1 # Number of instances
      register: ec2 # Store result in variable for later use
```
This creates a virtual machine in AWS.

## How It Works

```bash
Ansible → Cloud API → Cloud Provider → Resource Created
```

## Real DevOps Use Cases

- Provision EC2 instances
- Create VPC, subnets, security groups
- Deploy apps on cloud
- Auto-scale infrastructure
- Configure servers after creation

- Infrastructure as Code (IaC)
