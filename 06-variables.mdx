## Variables

Variables make playbooks **dynamic and reusable.**

- The concept of using variables is, in future if i want to change the tool name, i can only change the variable assigned to the value. This is to simplify the task
- Variable is anything that holds a value

**types of variables:**

- Static		- defined inside the playbook
- Dynamic	- not defined inside the playbook

1. Static variables defined inside the playbook:

```yaml
- hosts: all
  vars:
    a: git
    b: maven
    c: docker

  tasks:
    - name: installation of git
      yum: name={{a}} state=present

    - name: installation of maven
      yum: name={{b}} state=present

    - name: installation of docker
      yum: name={{c}} state=present

```

Command to run the playbook:

```bash
ansible-playbook install_tools.yml -e "a=git b=maven c=docker"
```

-e is used to pass extra variables while running the playbook. This is useful when we want to override the variables defined inside the playbook.

2. Dynamic variables not defined inside the playbook:

```yaml
- hosts: all
  tasks:
    - name: install package
      yum: name={{d}} state=present
```

Command to run the playbook:

```bash
ansible-playbook install_tools.yml -e "d=nginx"
```
