## Dynamic Inventory (☁️ MUST for Cloud)
👉 What it is:

Dynamic inventory means automatically fetching server details (IPs, hosts) from external sources like AWS, Azure, GCP instead of manually writing them.

### Problem:

- If new server is created → you must update manually ❌
- If server is deleted → inventory becomes wrong ❌

### 🧠 Static inventory (old way):

```bash
[web]
192.168.1.10
192.168.1.11
```

### 🧠 Dynamic (AWS example):

Ansible talks to cloud APIs and fetches servers automatically

### Example: AWS Dynamic Inventory
1. Install required packages:

```bash 
pip install boto boto3 # AWS SDK for Python
```

2️. Create config file: `aws_ec2.yml`

```yaml
plugin: aws_ec2
regions:
  - ap-south-1

filters:
  tag:Name: web-server
```
3️. Fetch inventory:

```bash
ansible-inventory -i aws_ec2.yml --list
```

-i → inventory file

--list → list hosts in JSON format

🧾 Output:

```bash
{
  "web": ["10.0.0.5", "10.0.0.6"]
}
```

## 🔥 How it works internally
- Ansible uses plugin (aws_ec2) [what is plugin? A reusable code that extends Ansible's functionality]
- Plugin calls AWS API
- Fetches instances based on filters
- Builds inventory automatically

## 🚀 Real Use:
- Auto-detect EC2 instances
- Works with autoscaling
- No need to update inventory manually
