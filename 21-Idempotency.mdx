## Idempotency

👉 What it is:

Running the same playbook multiple times gives the same result without making unnecessary changes.

### Real-Life Meaning

Imagine:

You install nginx once ✅
Run playbook again → it should NOT reinstall ❌

👉 That is idempotency

### ❌ Bad Example:

```bash
- name: Run command
  command: apt install nginx -y
```

### Problem:
- Runs every time
- Not aware of current state
- Can break things

✅ Good Example:

```yaml
- name: Install nginx
  apt:
    name: nginx
    state: present
```

👉 Runs only if not installed ✔️

🧠 Output:

```bash
first run → changed
second run → ok (no change)
```
### 🚀 Real Use:
- Safe deployments
- Avoid breaking systems
- Consistent infrastructure
